我的个人数据备份之路

2023-06-07
数据分析
字数统计: 1.1k   |   阅读时长≈ 3 分钟

整理一下我的个人数据存储和备份方法的历程。

目的

其实我不是单纯地涉及备份，还包括资料如何存储，如何获取的问题。因为我的目的主要是两个，第一是防止数据丢失，这是最基本的目的；第二就是我想要获取相应的资料时需要容易获取容易找到，因为存储的目的是为了使用，如果不方便使用那就也没有价值了。

数据分类

我目前的数据应该可以主要分为三类：

  1. 往期数据：主要是研究生，本科的东西，多且杂，利用频率较低。
  2. 工作数据：工作电脑上的东西
  3. 个人文档：自己整理的文档，基本是 markdown 格式。

下面基于每一块数据进行讨论。

备份方法

往期数据

这个最简单，百度网盘 + 移动硬盘。

因为目前利用频率低，需要的时候再去找就行，查找多花点时间也没事。

工作数据

我的习惯是将所有东西放在桌面，然后将桌面移动到 D 盘（防止系统损坏，需要重装电脑）。

然后在桌面上建立相应的文件夹，每周对其中重要的文件夹进行打包压缩，然后压缩包再上传到服务器。这样可以说是三备，相同的数据存放在个人电脑未压缩的文件夹，个人电脑里的压缩包（放在了另一个磁盘里），服务器里的压缩包三个地方。只要我的工作电脑和服务器不是同时损坏（这应该是非常小概率的事件），我最多就损失一周的工作量。

个人文档

这里又可以细分为两块，短文档和长文档。

短文档

对于短文档，我目前最终使用的解决方案是 hexo + onedrive 。

我这个网页就是 hexo 搭建的个人博客，将文档发布在我的这个博客上，这样只要有网络就可以访问，很方便。但是还有一个小问题就是，我在写新的文档时可能需要复制之前文档中的公式（因为不想自己重新打公式），但是我是无法从网页上复制公式的，我还需要去获取原始的 markdown 文件。

因此我进一步使用了 onedrive ，我一开始是将这个博客的东西都搬移到了 onedrive ，然后我发现由于同步不及时的原因， onedrive 容易产生一堆的副本文件，因此又把博客迁移回去了（汗！）。

后来在 B站视频 【神器】C盘拯救者 - FolderMove - 妈妈再也不怕我C盘不够用啦 发现可以通过符号链接的方式使用 onedrive 备份其它文件夹的内容。我就去官网 下载了 foldermove 软件，将博客的文件夹 _posts 移动了 onedrive 目录下（原位置会留下一个符号链接），然后更新博客内容， hexo 和 onedrive 就会保持同步，我在其它电脑上可以通过登录 onedrive来查看原始的 markdown 文件。

对于个人博客的全部原始文件，压缩后使用个人电脑+移动硬盘保存。

长文档

对于长文档，我目前使用的解决方案是 onedrive 。

我一开始也尝试过将长文档放在个人博客上，但是加载很慢，尤其是图片特别多的文档，于是便放弃了。

之后尝试过飞书，但是我的文档里往往包含图片，放在飞书上我需要将图片一个一个地黏贴过去，太麻烦了，于是也放弃了。

最终就还是用 onedrive 保存，不需要做什么修改，只需要图片使用相对路径，然后用 typora 正常打开就行了。

总结

  1. 硬盘有价，数据无价，重要数据的备份意识不可少。

  2. 除了备份以外，也要考虑自己获取这些数据的难易程度。

    按照我使用的这些方式来看，获取程度从易到难排名的话，首先是 hexo 博客最容易（条件只是网络），然后是 onedrive 和百度网盘 (网络+一台电脑) ，然后是个人电脑（因为我使用了远程软件，远程的话需要这台电脑保持开机状态，以及知道远程软件的账号秘密），最后才是移动硬盘。
