在当前目录下会生成很多文件夹和文件，其他都不用管，我们需要的就是 dist 目录下的 split_plink.exe 程序文件。

这里如果不使用 -F 选项，那么 dist 目录下除了相应的 exe 文件还会生成很多别的文件，运行 exe 程序时，必须同时包含dist 文件夹下的其他文件，只有 exe 程序 运行不成功。我们使用了 -F 选项 ，就只有一个 exe 程序 ，拷贝程序只要拷贝这个 exe 程序即可，更加方便。

源代码修改

我们修改了代码 split_plink.py ，主要在于以下几点：

  1. 固定文件名：原来通过输入参数，现在不再使用参数，固定输入文件名。

  2. 复制文件从 cp 改为 copy ，因为要从 linux 系统改为 windows 系统。

  3. 所有退出程序的地方，新增一条input 命令。之所以要这么做，是因为你在 windows 下双击 exe 程序，无论是运行成功或失败，运行窗口都是一闪而过，因此我新增了一个输入语句，让用户自己随便按键才结束运行窗口。

  4. 代码功能上，新增了一个需要剔除的芯片号文件，exclude_chipid.txt ，该文件中的芯片号不会出现在结果文件中。

运行程序

将输入文件 all.ped, all.map, split_plink.txt, exclude_chipid.txt 和运行程序 split_plink.exe 放在同一个文件夹下，双击exe文件，如果输入文件没有问题即可得到拆分结果文件。
